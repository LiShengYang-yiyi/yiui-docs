# 异步 无限循环列表

 
![Image](/images/CB5ewIAQZiBf1Ykp2DDcMj22nHb_0_81cbc665.png)
  针对需要异步无限循环列表的需求 
 
 💡 异步肯定会带来写法上的改变 还需要自己注意 
 
 同步与异步方案 二选一 
 
 如果都想要的可以自己修改源码 把其中一个改个名字或者命名空间来区分达到同时使用 
 我这里就不提供了 请自行解决 
 
 框架中将会同时存在2套代码 使用Unity文件夹功能  使用 [ . ] (文件夹前缀 "."(点)) 
 来实现忽略文件夹 不参与编译与显示 
 
![Image](/images/ZnTKwUVq0ir210ko6vPcSnSsnhM_1_b2af32fe.png)
 打开文件夹 多了一个 .Async 文件夹的标识 这是异步方案 (文件夹没有实际作用 仅仅是为了区分而已) 
 
![Image](/images/ZnTKwUVq0ir210ko6vPcSnSsnhM_2_dea8f0f2.png)
 (为什么没有文件夹名字来区分 为了保持代码结构 等原因...) 
 
 切换方法 
 2个文件夹换名字即可 
 
 FAQ: 为何没有写个工具点一下自动切换呢? 
 因为没必要  方案二选一 并没有经常切换的需求 请接入框架时提前选择好 
 如果有需要请自行实现 
 
 注意 
 为了保证之前的预制挂载的脚本不失效 请保证前后GUI一致 
 也就是meta文件一致 
 
 参考方案: 
 假设你现在工程中使用的是 同步方案 现在想改为异步方案 
 
 1 换文件夹名字  (使用异步的文件夹  同步的忽略) 
 2 将同步文件夹(你之前使用的方案) 中的所有.meta 文件 拷贝出来 替换掉 现在异步方案中的所有.meta 
 
 其他 
 至于切换完毕后  你想保留2套代码 还是删除不用的那个 都自行决定
