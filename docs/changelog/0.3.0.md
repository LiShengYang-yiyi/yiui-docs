 0.3.0 
 
 1. 新增IYIUIWindowClose 接口 
 ◦ 你可以理解为OOP中的 子类重写Close方法 先触发base.Close() >> IYIUIClose 
 ◦ 然后触发现在的WindowClose事件 
 ◦ 切记如果base.Close()中吧自己给摧毁了，那么这里就不会触发了 
 ◦ 一个YIUIWindowComponent 可以重写多个ViewClose 但是无法保证先后顺序 
 ◦ 实现方式为 在对应YIUIWindowComponent中 AddComponent你的重写组件 
 ◦ 然后实现IYIUIWindowClose接口 
 ◦ 参考 TipsViewComponent 
 ◦ 有2种触发时机 默认关闭动画后触发 也可以动画前触发 
 
 1. 非Panel的View的close 可以触发IYIUIClose 接口
