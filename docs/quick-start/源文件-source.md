功能
1.
用来记录Panel结构的源文件, 自动生成工具
根据结构自动生成对应的Panel View
否则就要自己一个一个的手动生成
你也可以理解为生成器
2.
因为不想有人在Panel上直接操作结构
因为遇到过有人改Panel 导致的问题
拆分后在Panel上的相关功能都是锁定的
想要修改必须回到源文件
3.
源文件还可作为所有View的可视化预览
假设一个Panel 有4个页签 结构相互独立的情况下
你要如何同时看到4个页签 比如有时候看看位置有没有对齐 什么的
常规情况下你就只能打开Panel  然后把页签一个一个的拖进去预览
预览完毕过后又要删除掉 特别麻烦 源文件可以起到一个预览作用
假设没有源文件 上述功能就要考虑一下是否有必要性
如果觉得源文件麻烦上述功能也不是很重要
可以考虑 修改Panel源码 吧那几个只读字段打开即可
这样就可以在Panel上直接修改结构
使用
源文件就是拿来初始化用的
不要在源文件上做绑定操作功能什么的
新建一个Panel 定好结构就直接拆分  之后就没他什么事了 可以无视
重构
如果你到后期想修改结构时 可以先逆向panel 就可以生成一个新的源文件
在这个新的源文件上修改结构后重新导出即可
逆向方法 选中想逆向的Panel  >> 右键 >> YIUI >> 逆向Panel
(注意逆向是覆盖操作 记得保存源文件 如果有问题还能还原)
(有版本管理的无视)
已知问题:
逆向会打破多层嵌套的预制体的关联
所以逆向过后检查一下 关联是否正确
如果不正确的需要手动调整一下
