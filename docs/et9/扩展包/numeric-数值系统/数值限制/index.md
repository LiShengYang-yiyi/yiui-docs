# 数值限制

 
![Image](/images/NVlVwoDPDiJ8BMkPRO8cL2rznEe_1_51049aec.png)
 
 某个值被修改时 必须满足限制 
 如常见的 当前血量 要 >= 0  <= 最大血量 
 
 如果不是内部做好限制 就需要在修改的地方 设置值时判断修改 
 这样会造成很多冗余分散等问题 
 如 造成伤害 每次都要去判断一下 如果这次伤害比当前血量大 那么只能造成当前血量的最大伤害 
 如果有数值限制功能  以后你可以无脑扣血 不管你如何扣他都不可能低于0 
 
 
 ID = 限制的数值类型 
 可以是任意存在的数值 
 
 最小值 与 最大值限制 
 分数字限制 与 数值限制 
 
 数字 
 如当前血量不能低于0  所以这里 最小限制使用 数字类型 
 数值 
 限制的是当前数据中某个数值的当前值  如当前血量的最大值 <= 最大血量的最终结果 
 
 优先级 
 当  A 被 B 限制  B 又被 C限制时 
 那么就需要先初始 C 然后 B 最后A 
 但是配置的时候如果不是按咋个顺序来的就会错误 
 所以需要根据优先级 当一个配置中有一连串依赖时 
 根据优先级初始化 从小到大 
 
 正常情况下不建议多层限制 
 最好只限制一层(也就是不要用优先级) 
 
 案例 
 最大血量=400 
 当前血量=300 
 修改为600时 触发限制 最终结果为400
