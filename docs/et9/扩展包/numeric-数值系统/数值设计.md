# 数值设计

 
遇数字皆可数值 数值系统属于非常底层的系统功能 
 如果把本套数值系统活用 
 一些有关于数字的系统用上数值系统会更加的高效 
 且各系统之间配合更好 
 数值系统是一个不光用于技能的,可以在任何系统上使用 
 数值属于框架级别的基础系统 
 
 为什么要独立 没有直接覆盖NumericComponent 
 像Luban那样直接替换Excel  不过不太一样这个可以共存 
 所以根据自己的需求 想一起用就一起用  也可以单独用 
 
 1.为什么数值系统可以挂在任何组件下 而不是限制 Unit 
 2.有了NumericDataComponent，为什么要里面包一层NumericData 
 3.为什么会有 相加 相减 的API 
 
 上述问题一起回答 遇数字皆可数值 
 
 以一个传统MMO 系统举例: 
 如果现在有个装备系统 肯定会涉及到装备组件 
 请问装备组件上有各种属性怎么做? 
 角色属性怎么做? 
 角色穿戴装备 属性整合怎么做? 
 
 我现在没有限制数值系统做法就很简单了 
 装备组件直接挂上数值 
 角色挂上数值 
 完事  要得到穿戴属性更简单 都不需要我计算 相关数值系统相加即可... 
 是不是很多事情都变的简单了... 
 
 这里是不是就算回答了 1 3问题 
 不限制数值系统可以挂在指定组件下 可以最大程度的设计 
 遇数字皆可数值 
 玩家的 金币 钻石  不会也自己独立写字段吧 
 这个为什么不能数值系统   player下面挂数值 
 就包含玩家的货币 都在里面了 这一下连同步什么的都不需要额外做了 
 数值系统本来就有同步 还有存储 
 
 玩家的设置保存 是不是也能用数值系统 
 不管是存储玩家当前的音量? 某个系统的开关? 等等 都可以用数值存储 
 取也方便 存也方便 同步什么都有了 
 
 比如常见的 天赋系统 强化系统  都可以加强玩家的数值 
 怎么做? 难道真的天赋系统 强化系统 额外写字段来存 攻击力+多少??? 
 现在还不简单 天赋系统挂上数值  强化系统挂上数值 
 当需要计算 玩家+天赋+强化系统 的最终收益时  只需要简单的数值相加就可以得到 
 这就是为什么要自由的挂载  以及相加相减 
 
 回到2号问题上 
 为什么要包一层Data 没有直接在组件上用字典 
 因为考虑到数值快照拷贝备份等这种同时多数据存储在同一个地方时 
 组件是无法解决复杂结构的 
 
 比如技能系统 脱手技能  假设脱手技能需要快照施法者的数值 
 传统做法 是不是得给这个脱手技能挂在一个数值系统作为快照 
 但是这个脱手技能也有自己的数值呢?  那是不是就得考虑加个额外的叫做快照组件的组件挂上数值?
