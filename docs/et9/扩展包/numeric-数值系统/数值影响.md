# 数值影响

 
这样的自由度最高 可以1对多的影响 也不用限制影响公式 
 但是 这仅限于 独立开发 如果是团队开发的情况下 
 💀 没有深入了解不懂的情况下 不建议使用此功能 
 
 个人还是更愿意把这些开放给策划 至少某些情况下策划可以自行调整 
 而不会因为某个小小的改动就要找程序 
 引入公式系统 TODO 
 
 根据配置 自动生成公式代码

 从结果上来看 还是在程序方面实现了对应的功能 
 优点 
 1. 这个是底层上去做的数值影响  跟使用NumericChange这个事件有本质区别 
 2. 程序省心 扩展度 自由度更高 
 3. 使用配置 策划可以没有程序介入的情况下扩展功能 
 (不要说什么策划水不会写代码...  那都不是问题 数值策划会的你相信我) 
 
 缺点 
 要在配置表裸写代码要求是高一点 (但是也就高一点) 
 简单的加减乘除那种很简单的  如果对于复杂的需求 第一版可以考虑在编辑器里写拷贝过去就好了 
 以后要修改某个值的大小时 就可以交给策划了
