 环境安装与配置 
 • ET主页访问 ：首先访问ET的官方网站，根据运行指南逐步操作。 
 • 安装环境 ：按照当前环境要求安装必要的软件和工具，如Unity版本和翻墙工具。 
 • Unity工程设置 ：在Unity工程中进行特定设置，包括管理配置和包管理。 
 GitHub Token的获取与使用 
 • 获取Token ：在GitHub中创建一个新的Token，选择永久有效期，并勾选所有权限。 
 • Token用途 ：Token用于包开发，以便在获取包时进行身份验证。 
 包管理和安装 
 • Unity包管理 ：在Unity中通过配置文件添加状态同步包，或使用package manager添加。 
 • 安装方式 ：大批量安装时推荐使用配置文件方式安装，因为package manager方式失败概率较大。 
 项目设置和初始化 
 • ET loader配置 ：在Unity中设置ET loader，确保使用正确的Token。 
 • 编译和初始化 ：编译项目并进行初始化，包括移动包和刷新Unity缓存。 
 分支管理和更新 
 • Git分支操作 ：创建和管理Git分支，包括状态同步分支和YIUI分支。 
 • 忽略文件夹设置 ：调整Git忽略文件夹设置，以便更好地管理项目文件。 
 • 奥丁安装 ：安装奥丁（Odin）插件，用于包管理和自动化更新。 
 包更新流程 
 • 检查更新 ：使用ET的包管理工具检查和更新包。 
 • 更新操作 ：演示如何更新包，包括处理更新冲突和重新编译项目。 
 教程结尾 
 • 分支推送 ：将完成的分支推送到远程仓库。 
 • 下步指导 ：提供关于如何继续进行Demo开发的指导。 
 总结 
 该指南为开发者提供了一个清晰的流程，从环境安装到项目配置，再到包管理和更新，确保开发者能够顺利地在ET9.0项目中集成状态同步包。同时，还涵盖了Git分支管理和奥丁插件的使用，为开发者提供了一套完整的工具和流程，以便更高效地管理和更新项目。 
 
 
 二 YIUI框架概述 (15分钟) 
 
 (纯理论 也可以跳过直接看后面) 
 • 名称由来 ：YIUI名称取自“亦”字，谐音“易”，旨在成为一个易于使用且简单的框架。 
 • 设计理念 ：结合多年工作经验和多个项目的实战总结，特别适用于大型项目，强调UI表现层与数据层的分离，提高开发效率和代码清晰度。 
 
 核心特性 
 1. 分离设计 ：将UI表现层与数据层进一步分离，逻辑层不依赖于具体的UI组件，数据变化驱动UI更新。 
 2. 简化使用 ：减少逻辑层的复杂性，界面的打开、关闭、回退、加载等操作由框架管理。 
 3. 分布式设计 ：支持多层UI管理，适应复杂的项目结构。 
 4. 分块式加载 ：按需加载UI组件，提高性能和效率。 
 5. 多样化打开方式 ：支持多种参数和方法打开UI，包括无限参数和配置文件触发。 
 6. 自适应设计 ：支持自适应刘海屏等现代设备特性。 
 7. 生命周期管理 ：参考MonoBehaviour的生命周期，为UI组件提供完整的生命周期管理。
