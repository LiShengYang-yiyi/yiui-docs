# 更新 (测试功能)

 
![Image](/images/CB5ewIAQZiBf1Ykp2DDcMj22nHb_0_81cbc665.png)
  测试功能 不保真 
 测试过大部分可以更新但是需要注意 
 
![Image](/images/HCGxwL9Kmi2Zr4kqXq5cine9nBb_1_2b6faf7a.png)
 当查询到网上有更高的版本时 则会提示更新 
 目前功能还无法保证所有包都可以更新 验证了部分包都没有问题 
 (有可能有些包里更新过后会出问题) 
 
 逻辑流程: 
 会删除当前包 且吧对应依赖这个包的版本号都修改成最新的版本 
 保存Unity会重新编译  此时会发现依赖的包不存在 则会自动下载到指定版本号 
 
 但是这个时候包被下载后再Cache文件夹下 还需要移动包文件到Package文件下 
 并且执行ET相关操作  比如重新生成相关引用程序集等等... 
 
 1. 使用移动文件指令  但是也有可能因为报错 看不到了 等等情况... 
 
![Image](/images/HCGxwL9Kmi2Zr4kqXq5cine9nBb_2_45296784.png)
 
![Image](/images/HCGxwL9Kmi2Zr4kqXq5cine9nBb_3_d9674a4a.png)
 2. 手动移动 
 找到对应的包移动到指定目录下 名字改成包名 
 然后还是要执行ET相关必要操作 
 
 3. 配置重新生成 
 已知某些升级后需要重新生成配置  所以可能需要手动刷新一下  以后考虑增加这个功能
